## API Keys

All API requests require an API key passed via the `Authorization` header using Bearer token authentication.

```bash
Authorization: Bearer tt_live_your_key_here
```

### Key types

| Prefix | Environment | Description |
|--------|-------------|-------------|
| `tt_live_` | Production | Full access to your real data |
| `tt_test_` | Development | Sandbox mode — no real data affected |

### Generate a key

<Steps>
  <Step title="Open Settings">
    In the Take Time app, navigate to the **Settings** tab (gear icon).
  </Step>
  <Step title="API Keys section">
    Scroll to **Developer** and tap **Generate API Key**.
  </Step>
  <Step title="Copy and save">
    Copy the key immediately — it will only be shown once. Store it securely.
  </Step>
</Steps>

<Warning>
  Never expose your API key in client-side code, public repositories, or browser requests. Always use it server-side or in secure environments like MCP server configs.
</Warning>

## Rate Limits

| Plan | Requests/min | Requests/day |
|------|-------------|-------------|
| Free | 30 | 1,000 |
| Pro | 120 | 10,000 |

When rate limited, the API returns `429 Too Many Requests` with a `Retry-After` header.

## Permissions

Each API key has full access to the owner's data. There are no granular permission scopes at this time.

<Info>
  Scoped permissions (read-only keys, per-resource access) are planned for a future release.
</Info>

## Using with MCP

When configuring the MCP Server, pass your API key as an environment variable:

```json
{
  "mcpServers": {
    "taketime": {
      "command": "npx",
      "args": ["@taketime/mcp-server"],
      "env": {
        "TAKETIME_API_KEY": "tt_live_your_key_here"
      }
    }
  }
}
```

The MCP Server reads `TAKETIME_API_KEY` from the environment and authenticates all requests automatically.
