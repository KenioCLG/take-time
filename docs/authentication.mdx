## App Authentication

Take Time uses **Supabase Auth** with email and password. When you sign up at [taketime.space](https://taketime.space), your account is created in Supabase and your data is stored securely with Row Level Security (RLS).

Your data syncs between the app (local storage) and the cloud automatically.

## MCP Server Authentication

The MCP Server supports two authentication methods:

<Tabs>
  <Tab title="Email/Password (Recommended)">
    Use your Take Time account credentials. The session **auto-refreshes** — the server runs indefinitely without re-authentication.

    ```json
    {
      "env": {
        "TAKETIME_EMAIL": "your@email.com",
        "TAKETIME_PASSWORD": "your-password"
      }
    }
    ```
  </Tab>
  <Tab title="Access Token">
    Use a Supabase access token extracted from your browser session. **Expires after ~1 hour** — suitable for quick testing only.

    ```json
    {
      "env": {
        "TAKETIME_API_KEY": "your_supabase_access_token"
      }
    }
    ```

    <Warning>
      Access tokens expire. For persistent use, always prefer email/password authentication.
    </Warning>
  </Tab>
</Tabs>

## Security

- Credentials are passed as environment variables in your MCP config — they are never stored in the codebase
- All communication with Supabase uses HTTPS
- Row Level Security ensures users can only access their own data
- Add your MCP config files to `.gitignore` if they contain credentials

<Info>
  There is no REST API or API key system (`tt_live_`, etc.). All programmatic access goes through the MCP Server using your account credentials.
</Info>
