## Overview

Webhooks allow you to receive HTTP POST notifications when events occur in your Take Time account. This is useful for building integrations, dashboards, or automations.

## Events

| Event | Description |
|-------|-------------|
| `block.created` | A new study block was created |
| `block.completed` | A block was marked as done |
| `block.deleted` | A block was deleted |
| `subject.created` | A new activity was created |
| `subject.deleted` | An activity was deleted |
| `streak.milestone` | User hit a streak milestone (7, 30, 100 days) |

## Payload Format

```json
{
  "event": "block.completed",
  "timestamp": "2026-06-11T10:00:00Z",
  "data": {
    "block_id": "blk_abc123",
    "subject_name": "JavaScript",
    "date": "2026-06-11"
  }
}
```

## Setup

<Steps>
  <Step title="Register your endpoint">
    In the app Settings, go to **Developer > Webhooks** and add your URL.
  </Step>
  <Step title="Select events">
    Choose which events you want to receive.
  </Step>
  <Step title="Verify signature">
    Each webhook includes an `X-TakeTime-Signature` header. Verify it using your webhook secret.
  </Step>
</Steps>

## Signature Verification

```javascript
const crypto = require('crypto');

function verifyWebhook(payload, signature, secret) {
  const expected = crypto
    .createHmac('sha256', secret)
    .update(payload)
    .digest('hex');
  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(expected)
  );
}
```

<Warning>
  Always verify webhook signatures before processing the payload. Never trust unverified webhooks.
</Warning>
