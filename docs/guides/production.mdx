## Recommended Setup

<Steps>
  <Step title="Use email/password auth">
    Always prefer email/password over access tokens. Email/password sessions auto-refresh, while access tokens expire after ~1 hour.
  </Step>
  <Step title="Secure your credentials">
    Store credentials in environment variables via your AI client's MCP config. Never commit config files with passwords to version control.
  </Step>
  <Step title="Use absolute paths">
    In your MCP config, use the full absolute path to `src/index.js`. Relative paths may not resolve correctly depending on your AI client.
  </Step>
  <Step title="Keep the server updated">
    The `npx` command always fetches the latest version. To update a global install:

    ```bash
    npm install -g @taketimemcp/mcp-server@latest
    ```
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Tools not appearing">
    - Verify the path to `src/index.js` is absolute and correct
    - Check that `node` is in your system PATH
    - Make sure `npm install` was run in the `mcp-server/` directory
    - Restart the AI client completely (not just the conversation)
  </Accordion>
  <Accordion title="Authentication errors">
    - Verify your email and password are correct
    - Make sure you have a Take Time account at [taketime.space](https://taketime.space)
    - If using access token, note it expires after ~1 hour
  </Accordion>
  <Accordion title="Connection timeouts">
    - Check your internet connection
    - The server needs to reach `gwujukwvufmfecqgzvrs.supabase.co`
    - Try running `node src/index.js` directly to see error messages
  </Accordion>
</AccordionGroup>
